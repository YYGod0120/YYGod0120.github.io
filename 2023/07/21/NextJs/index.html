什么是 Next.js

拖了好久的 Next.js 终于有空来学一学了

按照惯例，先介绍一下 Next.js，它是一个 react 的框架，一个框架的框架。当你觉得写 react 在某些时候很不方便的时候，就有必要来学习一下 Next.js 了

它的优势在于：

  • 既支持 SSG(静态预渲染页面),也支持 SSR(请求时预渲染页面)
  • 基于文件下的路由
  • 内置支持 CSSModule,Sass,tailwindcss
  • 支持 Typescript
  • 0 配置，快速刷新，代码拆分和打包

SSG 与 SSR

SSG 指的是页面预渲染，当我们使用 react 构建项目的时候，都是通过执行 js 代码来生成 html 和样式然后渲染到页面上，当你对一个 react 项目禁用 js 的时候，你就会发现他们打不开了，而 Next.js 则会提前渲染每个页面的 html，就算你禁用了 js，他一样会被渲染出来。

Next.js 实现预渲染有两种方式,一个是静态生成(客户端预渲染),一个是请求时生成(后端预渲染)。

如果你需要在请求时渲染页面，那就选择 SSR.如果时请求前渲染，那就选择 SSG

基于文件下的路由

一般我们编写 react 的时候离不开路由的切换，而路由的切换就要使用到 react-router-dom 来写，有一点小小的繁琐，而 next.js 支持基于文件的路由，也就是当你在 pages 目录下编写文件的时候，该文件目录就会直接变成 URL 路径。
比如说:假设你在/pages/about.js路径下写了一串代码，那么访问http://localhost:3000/about就会呈现出你的页面(本地开发)

支持 CSSModule，Sass，Tailwindcss

Next.js 框架支持 CSSModule，Sass，Tailwindcss。
CSSModule 就是将 Css 模块化避免冲突。在 Next.js 中将你 css 的后缀改为 xxx.module.css 使用import styles from './layout.module.css'并且在需要用到 style 的地方styles.name来使用
其他的两种需要的时候可以自己查阅文档

Next.js 提供一些组件

<Link>组件支持在 pages 之间跳转，类似Link的组件。路径就是pages/
<Img>组件更加优化你的图片的生成，它自带懒加载以及图像优化
<Hea>组件允许你修改页面的<head>
<Script>组件支持高性能的导入第三方 script 脚本。

  • strategy 控制何时加载第三方脚本。 lazyOnload 值告诉 Next.js 在浏览器空闲期间延迟加载此特定脚本
  • onLoad 用于在脚本完成加载后立即运行任何 JavaScript 代码。在此示例中，我们将一条消息记录到控制台，指出脚本已正确加载

如果需要知道更多 Next.js 的内置组件以及组件的用法，可以查阅官方文档
